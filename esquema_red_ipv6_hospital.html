Esquema lógico de red IPv6 — Hospital General de Salud
Plan de direccionamiento para las 6 subredes diferenciadas · Implementación en Packet Tracer
Prefijo asignado (documentación RFC 3849): 2001:db8:: /32
Prefijo de sitio usado: 2001:db8:cafe::/48 — divide en subredes /64 (18 trillones de hosts por subred)
Máscara por subred: /64 · Gateway de cada subred: primer host ::1
1
Servidor web (HCE)
2001:db8:cafe:1::/64
Servidor de consulta de historias clínicas electrónicas. Página web con nombre del hospital e integrantes del grupo. Solo comunicación IPv6.
Servidor HTTPGateway ::1
2
Red inalámbrica (Wi-Fi)
2001:db8:cafe:2::/64
Punto de acceso Wi-Fi para personal médico móvil. Tablets y smartphones configurados con IPv6. Movilidad dentro del hospital.
Access PointTablet médicoSmartphone
3
Servidor de correo
2001:db8:cafe:3::/64
Servidor de correo con dominio @hospital.com. Comunicación interna entre médicos, administrativos y especialistas sobre IPv6.
Servidor SMTP/IMAP@hospital.com
4
Usuarios locales (consultorios)
2001:db8:cafe:4::/64
Estaciones de trabajo de médicos y personal administrativo. Acceden a servicios web y correo. Direccionamiento IPv6 estático.
PC MédicoPC AdministraciónImpresora
5
Sede principal (especialistas)
2001:db8:cafe:5::/64
Infraestructura de la sede capital con médicos especialistas. Enlazada al hospital por router IPv6 para garantizar escalabilidad y compatibilidad futura.
PC EspecialistasRouter WAN
6
Enlace router (telemedicina)
2001:db8:cafe:6::/64
Subred de interconexión entre router local del hospital y router de sede principal. Simula la red WAN de telemedicina. Tráfico exclusivamente IPv6.
Router local ::1Router sede ::2
